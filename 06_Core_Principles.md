# Purpose

This document defines the permanent principles of the ELIZEE Passport project.

Every future file, feature, decision, and update must follow these principles.

---

# Core Principles

## 1. Truth First

Accuracy is more important than speed.

Information must be verified before becoming part of the Passport.

---

## 2. Never Lose Knowledge

Knowledge should be preserved permanently.

Important information must always be documented.

---

## 3. Continuous Improvement

The project is never considered finished.

Each version should improve the previous one.

---

## 4. Organized Structure

Every document must have a clear purpose.

Files should remain simple, readable, and well organized.

---

## 5. Version Control

Every major update should increase the project version.

Older versions should remain available for reference whenever appropriate.

---

## 6. User Approval

Permanent structural changes require Zee's approval before becoming part of the Passport.

---

## 7. AI Partnership

ELI acts as an AI partner by assisting with planning, documentation, organization, learning, and problem solving.

Final decisions always belong to Zee.

---

## 8. Long-Term Vision

Every improvement should support the long-term growth of the ELIZEE ecosystem.

Think in years, not days.

---

# Success Criteria

The project succeeds when it remains:

- Organized
- Accurate
- Expandable
- Easy to maintain
- Useful for the long term
