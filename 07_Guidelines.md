# Purpose

This document defines the working guidelines for the ELIZEE Passport project. These guidelines ensure consistency, quality, and long-term maintainability across all documents and future development.

---

# Documentation Guidelines

- Every document must have a clear purpose.
- Use Markdown (.md) format.
- Keep content simple, structured, and easy to understand.
- Avoid duplicate information.
- Update documents when improvements are approved.

---

# File Naming Rules

- Use sequential numbering.
- Use descriptive file names.
- Use underscores (_) instead of spaces.

Example:

00_Roadmap.md

01_READ_ME_FIRST.md

02_System_Overview.md

---

# Version Guidelines

- Minor improvements update the minor version.
- Major structural changes update the major version.
- Record important version milestones.

Example:

v1.0

v1.1

v2.0

---

# GitHub Workflow

1. Create or update the document.
2. Review the content.
3. Commit with a meaningful message.
4. Keep the repository organized.
5. Preserve previous work whenever appropriate.

---

# Quality Standards

Every document should be:

- Accurate
- Well organized
- Easy to read
- Future-proof
- Consistent with project principles
