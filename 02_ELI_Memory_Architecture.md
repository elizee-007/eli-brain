# Purpose

This document defines how ELI stores, organizes, retrieves, updates, and protects information throughout its lifetime. It establishes the foundation of ELI's long-term memory system.

---

# Memory Vision

ELI should remember only meaningful information that helps the user over the long term while avoiding unnecessary or temporary data.

---

# Memory Types

## Permanent Memory

Information that rarely changes.

Examples:

- Identity
- Core Principles
- Long-Term Goals
- Mission
- User Preferences
- Important Projects

---

## Working Memory

Information currently being used.

Examples:

- Active Tasks
- Current Conversations
- Ongoing Projects
- Temporary Notes

---

## Knowledge Memory

Structured information collected over time.

Examples:

- Research
- Learning Notes
- Documentation
- AI Knowledge
- Islamic Knowledge
- Engineering Knowledge

---

## Archive Memory

Completed or historical information kept for future reference.

Examples:

- Completed Projects
- Old Versions
- Finished Tasks
- Historical Decisions

---

# Memory Rules

- Store only valuable information.
- Avoid duplicate memories.
- Verify information before saving.
- Keep memories organized.
- Update outdated information.
- Protect user privacy.

---

# Memory Lifecycle

1. Capture
2. Verify
3. Organize
4. Store
5. Retrieve
6. Update
7. Archive

---

# Memory Sources

Build a reliable memory system that allows ELI to become increasingly helpful while preserving important knowledge for years.
